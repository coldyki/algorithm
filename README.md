프로토타입 설명  
GraphTrip은 사용자의 장소 저장/좋아요 데이터를 기반으로 여행 장소와 여행 코스를 추천해주는 서비스입니다.

초기 서비스 단계에서는 특정 장소를 좋아한 사용자들이 함께 좋아요/저장한 다른 장소를 추천합니다.

이후 사용자 데이터가 쌓이면, 리뷰 작성 시 선택한 키워드(감성적인, 조용한, 사진맛집, 가성비 등)를 활용해 사용자 취향에 맞는 장소와 여행 코스를 개인화 추천하는 방식으로 확장할 수 있습니다.

또한 추천된 장소들을 이동 거리 기준으로 최적화하여 하나의 여행 코스로 제공합니다.

Who  
- 여행 계획을 쉽게 세우고 싶은 사용자
- 자신의 취향에 맞는 장소를 추천받고 싶은 사용자
- 인기순이 아닌 개인 맞춤형 여행 코스를 원하는 사용자
- 처음 방문하는 지역에서 효율적인 동선을 추천받고 싶은 사용자

When  
- 여행을 계획할 때
- 주말 나들이 코스를 찾을 때
- 특정 장소 방문 후 비슷한 분위기의 장소를 찾고 싶을 때
- 여행 일정과 이동 동선을 효율적으로 구성하고 싶을 때

Why  
기존 여행 추천 서비스는 단순 인기순 추천이나 지역 기반 추천이 많아 개인 취향을 충분히 반영하지 못하는 한계가 있습니다.

본 서비스는 사용자 행동 데이터, 장소 간 연관성, 리뷰 키워드, 이동 거리 정보를 함께 활용하여 맞춤형 여행 추천을 제공하는 것을 목표로 합니다.

## 5. 사용 자료구조

Graph: 경주 여행지를 정점으로 표현하고, 장소 간 이동 거리를 간선의 가중치로 표현하기 위해 사용하였다. 여행 장소 간 연결 관계를 나타내는 데 적합하다.

Priority Queue: Dijkstra 알고리즘에서 현재까지의 거리가 가장 짧은 장소를 빠르게 선택하기 위해 사용하였다. Python의 heapq를 활용하였다.

Dictionary: 장소 ID를 기준으로 장소명, 키워드, 거리 정보를 빠르게 조회하기 위해 사용하였다. places.json을 읽은 뒤 place_id를 key로 저장한다.

Set: Greedy 알고리즘에서 아직 방문하지 않은 추천 장소를 관리하기 위해 사용하였다. 방문 여부 확인과 제거가 빠르고, 중복 장소를 방지할 수 있다.

List: 추천 장소 ID 목록과 최종 여행 코스처럼 순서가 필요한 데이터를 저장하기 위해 사용하였다.

## 6. 사용 알고리즘

Dijkstra Algorithm: 현재 위치에서 다른 장소까지의 최단 거리를 계산하기 위해 사용하였다. 장소 간 이동 거리는 모두 양수이므로 Dijkstra 알고리즘을 적용하기 적합하다.

Floyd-Warshall Algorithm: 모든 장소 쌍 사이의 최단 거리를 미리 계산하기 위해 사용하였다. 본 프로토타입은 경주 주요 여행지 10개를 대상으로 하므로, 전체 최단거리 테이블을 계산해도 부담이 작다.

Greedy Algorithm: 현재 위치에서 아직 방문하지 않은 장소 중 가장 가까운 장소를 다음 방문지로 선택하기 위해 사용하였다. 모든 방문 순서를 완전탐색하지 않고 빠르게 효율적인 코스를 생성할 수 있다.

Frequency Counting: 최종 코스에 포함된 장소들의 키워드 빈도를 계산하기 위해 사용하였다. 이를 통해 코스의 주요 키워드와 코스 유형을 출력할 수 있다.

## 12. 향후 확장 방향

현재 프로토타입은 경주 주요 여행지 10개를 기준으로 구현하였다.

Floyd-Warshall 알고리즘은 모든 장소 쌍의 최단거리를 계산하므로 장소 수가 적은 현재 프로토타입에는 적합하다.

그러나 향후 장소 수가 수백 개 이상으로 증가하면 Floyd-Warshall의 O(V^3) 시간 복잡도가 부담될 수 있다.

따라서 실제 대규모 서비스로 확장할 경우 다음 방식을 고려할 수 있다.

- Dijkstra Algorithm 기반 부분 최단경로 탐색
- A* Algorithm 기반 휴리스틱 경로 탐색
- 지역별 그래프 분할 방식
- 지도 API 기반 실제 이동 시간 반영

현재 프로토타입은 경주 주요 여행지 10개를 기준으로 구현하였다.

향후 장소 수가 증가하면 Floyd-Warshall 알고리즘의 O(V^3) 시간 복잡도가 부담될 수 있으므로, 실제 대규모 서비스에서는 Dijkstra 또는 A* 기반 탐색, 지역별 그래프 분할 방식 등을 고려할 수 있다.
