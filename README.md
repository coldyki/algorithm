# algorithm

[기말 프로젝트 주제 정리]

프로토타입 제목  
GraphTrip 서비스 프로토타입  
: 사용자 행동 및 키워드 기반 개인화 여행 코스 추천 서비스

팀명  
Graph Navigators

프로토타입 설명  
GraphTrip은 사용자의 장소 저장/좋아요 데이터를 기반으로 여행 장소와 여행 코스를 추천해주는 서비스입니다.

초기 서비스 단계에서는 특정 장소를 좋아한 사용자들이 함께 좋아요/저장한 다른 장소를 추천합니다.

이후 사용자 데이터가 쌓이면, 리뷰 작성 시 선택한 키워드(감성적인, 조용한, 사진맛집, 가성비 등)를 활용해 사용자 취향에 맞는 장소와 여행 코스를 개인화 추천하는 방식으로 확장할 수 있습니다.

또한 추천된 장소들을 이동 거리 기준으로 최적화하여 하나의 여행 코스로 제공합니다.

Who  
- 여행 계획을 쉽게 세우고 싶은 사용자
- 자신의 취향에 맞는 장소를 추천받고 싶은 사용자
- 인기순이 아닌 개인 맞춤형 여행 코스를 원하는 사용자
- 처음 방문하는 지역에서 효율적인 동선을 추천받고 싶은 사용자

When  
- 여행을 계획할 때
- 주말 나들이 코스를 찾을 때
- 특정 장소 방문 후 비슷한 분위기의 장소를 찾고 싶을 때
- 여행 일정과 이동 동선을 효율적으로 구성하고 싶을 때

Why  
기존 여행 추천 서비스는 단순 인기순 추천이나 지역 기반 추천이 많아 개인 취향을 충분히 반영하지 못하는 한계가 있습니다.

본 서비스는 사용자 행동 데이터, 장소 간 연관성, 리뷰 키워드, 이동 거리 정보를 함께 활용하여 맞춤형 여행 추천을 제공하는 것을 목표로 합니다.

###  팀원2 - 주요 구현 내용 및 기술적 의도

1. **Trie 자료구조 기반 접두사(Prefix) 검색 구현**
   - 여행지 키워드를 글자 단위로 분기하여 공통 앞부분(Prefix)을 공유하는 트리 구조 구축
   - 전체 데이터 규모와 무관하게 사용자 입력 단어 길이 $O(L)$만큼만 탐색하는 최고 수준의 검색 효율 달성
   - 자식 노드를 파이썬 동적 HashMap(`dict`)으로 구현하여 일반 Trie의 고질적인 메모리 낭비 문제를 보완

2. **HashMap 매핑을 통한 텍스트 데이터의 정수(Index) 화**
   - 자연어 상태인 키워드들을 컴퓨터가 연산할 수 있도록 고유한 숫자 방 번호(0~20)로 1:1 매핑 변환
   - `len(unique_keywords_list)`를 통해 전체 키워드 개수(`num_keywords = 21`)를 동적으로 파악
   - 데이터의 규모가 변하더라도 코드가 깨지지 않고 유연하게 대처할 수 있도록 확장성을 고려한 설계 적용

3. **TF-IDF 기반의 키워드 특징 벡터(Feature Vector) 추출**
   - 각 여행지 키워드의 희소성과 가치를 수학적으로 계산하여 21차원의 규격화된 벡터 생성
   - 흔한 단어(예: 산책)의 가중치는 낮추고, 고유한 단어(예: 문화유산)의 가중치를 높여 장소의 특징을 명확히 정량화
   - `collections.defaultdict`를 활용하여 조건문 예외 처리 없이 O(1) 속도로 안전하게 DF(Document Frequency) 집계

4. **순수 파이썬(From Scratch) 기반 K-Means 군집화 알고리즘 구현**
   - 외부 머신러닝 라이브러리(`scikit-learn`)에 의존하지 않고 알고리즘 내부 메커니즘을 직접 빌드
   - 유클리드 거리 측정 및 중심점(Centroid) 업데이트  과정을 직접 구현하여 학습 목적에 충실한 코드 작성
   - 모든 여행지의 벡터 차원 크기를 동일하게 맞춰주어 공간 상에서 데이터 간의 공정한 거리 연산 수행
   - `random.seed(42)` 설정을 통해 실행 환경과 상관없이 완벽한 '실험의 재현성' 확보

### 결과 요약
경주 명소 10개 데이터를 분석하여 최종적으로 [역사/문화], [감성/데이트], [액티비티/휴양]이라는 3가지($K=3$) 유저 취향별 카테고리로 완벽히 분류 및 추천 시스템의 기반 마련.
