# algorithm

[기말 프로젝트 주제 정리]

프로토타입 제목  
GraphTrip 서비스 프로토타입  
: 사용자 행동 및 키워드 기반 개인화 여행 코스 추천 서비스

팀명  
Graph Navigators

프로토타입 설명  
GraphTrip은 사용자의 장소 저장/좋아요 데이터를 기반으로 여행 장소와 여행 코스를 추천해주는 서비스입니다.

초기 서비스 단계에서는 특정 장소를 좋아한 사용자들이 함께 좋아요/저장한 다른 장소를 추천합니다.

이후 사용자 데이터가 쌓이면, 리뷰 작성 시 선택한 키워드(감성적인, 조용한, 사진맛집, 가성비 등)를 활용해 사용자 취향에 맞는 장소와 여행 코스를 개인화 추천하는 방식으로 확장할 수 있습니다.

또한 추천된 장소들을 이동 거리 기준으로 최적화하여 하나의 여행 코스로 제공합니다.

Who  
- 여행 계획을 쉽게 세우고 싶은 사용자
- 자신의 취향에 맞는 장소를 추천받고 싶은 사용자
- 인기순이 아닌 개인 맞춤형 여행 코스를 원하는 사용자
- 처음 방문하는 지역에서 효율적인 동선을 추천받고 싶은 사용자

When  
- 여행을 계획할 때
- 주말 나들이 코스를 찾을 때
- 특정 장소 방문 후 비슷한 분위기의 장소를 찾고 싶을 때
- 여행 일정과 이동 동선을 효율적으로 구성하고 싶을 때

Why  
기존 여행 추천 서비스는 단순 인기순 추천이나 지역 기반 추천이 많아 개인 취향을 충분히 반영하지 못하는 한계가 있습니다.

본 서비스는 사용자 행동 데이터, 장소 간 연관성, 리뷰 키워드, 이동 거리 정보를 함께 활용하여 맞춤형 여행 추천을 제공하는 것을 목표로 합니다.

## 1. 담당 역할

팀원4는 추천 시스템을 통해 선정된 경주 여행 장소들을 입력받아, 이동 거리 기준으로 효율적인 여행 코스를 생성하는 기능을 담당한다.

담당 기능은 다음과 같다.

- 장소 거리 데이터 정리
- 최단경로 알고리즘 구현
- 추천 장소 기반 여행 코스 생성
- 기존 추천 순서와 최적화된 코스의 이동 거리 비교
- 최적 코스의 구간별 이동 거리 출력
- 코스 주요 키워드 및 코스 유형 출력

## 2. 사용 데이터

### 2.1 장소 데이터

파일 경로:

data/places.json

장소 데이터는 경주 주요 여행지의 ID, 이름, 키워드 정보를 저장한다.

예시:

{
  "id": 1,
  "name": "황리단길",
  "keywords": ["감성적인", "사진맛집", "카페투어", "데이트"]
}

### 2.2 거리 데이터

파일 경로:

data/distances.csv

거리 데이터는 장소와 장소 사이의 이동 거리를 저장한다.

예시:

from_place_id,to_place_id,distance_km
1,3,1.1
1,2,1.8
1,10,1.3

## 3. 사용 자료구조

- Graph: 장소를 정점, 장소 간 이동 거리를 간선 가중치로 표현
- Priority Queue: Dijkstra 알고리즘에서 최단 거리 후보를 관리
- Dictionary: 장소 ID, 장소명, 키워드, 거리 정보 조회
- Set: 아직 방문하지 않은 추천 장소 관리

## 4. 사용 알고리즘

- Dijkstra Algorithm: 하나의 출발 장소에서 다른 장소까지 최단 거리 계산
- Greedy Algorithm: 현재 위치에서 가장 가까운 미방문 장소를 다음 방문지로 선택
- Floyd-Warshall Algorithm: 모든 장소 쌍 사이의 최단 거리 계산

## 6. 현재 테스트 입력

팀원1, 팀원2, 팀원3의 추천 결과가 아직 연결되지 않았으므로 현재는 더미 추천 장소 ID 리스트를 사용한다.

예시:

start_place_id = 1
recommended_place_ids = [1, 2, 3, 8, 10]

## 7. 팀원 코드와의 연결 방식

팀원1, 팀원2, 팀원3의 추천 결과가 완성되면 최종 추천 장소 ID 리스트를 팀원4 모듈에 전달하면 된다.

예시:

from route_optimizer import optimize_travel_route

result = optimize_travel_route(
    start_place_id=1,
    recommended_place_ids=[1, 2, 3, 8, 10],
    method="floyd"
)

다른 팀원들은 최종적으로 다음 두 값을 넘겨주면 된다.

start_place_id = 1
recommended_place_ids = [1, 2, 3, 8, 10]

## 8. 현재 모듈의 역할 정리

팀원1, 팀원2, 팀원3은 사용자의 행동 데이터, 키워드 데이터, 개인화 점수를 기반으로 추천 장소를 선정한다.

팀원4 모듈은 추천된 장소 목록을 입력받아 이동 거리 기준으로 효율적인 방문 순서를 생성한다.

전체 연결 흐름은 다음과 같다.

사용자 좋아요/저장 데이터
→ 키워드 기반 장소 분류
→ 개인화 추천 점수 계산
→ 추천 장소 ID 리스트 생성
→ 팀원4 여행 코스 최적화
→ 최종 여행 코스 출력

## 9. 향후 확장 방향

현재 프로토타입은 경주 주요 여행지 10개를 기준으로 구현하였다.

향후 장소 수가 증가하면 Floyd-Warshall 알고리즘의 O(V^3) 시간 복잡도가 부담될 수 있으므로, 실제 대규모 서비스에서는 Dijkstra 또는 A* 기반 탐색, 지역별 그래프 분할 방식 등을 고려할 수 있다.
